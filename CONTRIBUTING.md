# Contributing to Deploytual

First off, thank you for taking the time to contribute to **Deploytual**! 🚀

We welcome bug fixes, feature enhancements, documentation improvements, and new ideas that help make Deploytual better for everyone.

---

# How to Contribute

### 1. Fork the Repository

Click the **Fork** button on GitHub.

---

### 2. Clone Your Fork

```bash
git clone https://github.com/your-username/Deploytual.git
cd Deploytual
```

---

### 3. Create a Feature Branch

```bash
git checkout -b feature/your-feature-name
```

---

### 4. Make Your Changes

- Write clean, readable code.
- Follow the existing project structure.
- Test your changes before submitting.

---

### 5. Commit Your Changes

Use clear and descriptive commit messages.

```bash
git add .
git commit -m "Add detailed description of your change"
```

Example:

```bash
git commit -m "Add MongoDB connection support"
```

---

### 6. Push Your Changes

```bash
git push origin feature/your-feature-name
```

---

### 7. Open a Pull Request

Create a Pull Request against the **main** branch of the original repository.

Please include:

- A short description
- Screenshots (if UI changes)
- Steps to test
- Related issue number (if applicable)

---

# Code Style

## Frontend

- Use semantic HTML.
- Keep JavaScript modular and readable.
- Maintain the existing vanilla JavaScript structure.
- Reuse existing UI components whenever possible.

---

## Backend

- Follow **PEP 8**.
- Use FastAPI best practices.
- Add type hints where appropriate.
- Keep routes and services separated.

---

# Reporting Bugs

If you discover a bug:

1. Check existing issues first.
2. Open a new issue if it doesn't already exist.
3. Include:
   - Expected behavior
   - Actual behavior
   - Steps to reproduce
   - Screenshots (if applicable)
   - Environment details

---

# Feature Requests

Have an idea?

Open an issue describing:

- The problem
- Your proposed solution
- Any alternatives you've considered

---

# Development Tips

- Keep Pull Requests focused on a single feature or fix.
- Avoid large unrelated changes.
- Write meaningful commit messages.
- Update documentation when necessary.

---

# Thank You ❤️

Every contribution—whether it's fixing a typo, improving documentation, reporting a bug, or adding a new feature—helps make **Deploytual** better for everyone.

Happy coding! 🚀
